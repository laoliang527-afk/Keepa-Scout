启动时做的事

FastAPI 容器启动时，通过 lifespan 上下文管理器自动执行数据库初始化和 ETL 数据导入，全程自动化，容器就绪时 DB 已有数据。

keepa_client.py — Keepa 数据解析

 1 # Keepa stats.current[] index mapping (US / domain=1)
 2 AMAZON_SELLER_ID = "ATVPDKIKX0DER"
 3
 4 def parse_product(self, p: dict) -> dict:
 5     # BuyBox 价格：优先用 stats.buyBoxPrice（最可靠），fallback 到 current[0]
 6     stats_data = p.get("stats", {}) or {}
 7     buybox_cents = stats_data.get("buyBoxPrice")
 8     if buybox_cents is None or buybox_cents < 0:
 9         buybox_cents = _safe_idx(p.get("current") or [], 0)
10     result["buybox"] = _cents_to_dollars(buybox_cents)
11
12     # 销售排名：stats.avg[3] = 90天平均排名（最稳定），fallback 到 current[2]
13     stats_avg = stats_data.get("avg") or []
14     sales_rank = _safe_idx(stats_avg, 3)
15     if sales_rank is None or sales_rank < 0:
16         sales_rank = _safe_idx(p.get("current") or [], 2)
17
18     # Amazon BuyBox 占比：从历史推算（buyBoxSellerIdHistory 交替排列）
19     history = p.get("buyBoxSellerIdHistory", [])
20     seller_ids = history[1::2]                        # 取奇数位 = 卖家ID
21     amazon_count = sum(1 for sid in seller_ids if sid == AMAZON_SELLER_ID)
22     result["amazon_buybox_pct"] = round(100.0 * amazon_count / len(seller_ids), 2)

upc_normalizer.py — 核心逻辑

 1 def normalize_upc(raw: str) -> list[str]:
 2     digits = "".join(ch for ch in raw if ch.isdigit())  # 只保留数字
 3     if not digits:
 4         return []
 5
 6     # 11位 → UPC-E，展开为12位 UPC-A（按校验位规则展开）
 7     if len(digits) == 11:
 8         expanded = upce_to_upca(digits)
 9         return [expanded, digits] if expanded else []
10
11     # 12位 → 直接用，加前缀0得到 ITF-14
12     if len(digits) == 12:
13         variants = [digits, "0" + digits]
14
15     # 13位 → EAN-13，去前缀0得到12位 UPC-A
16     if len(digits) == 13:
17         variants = [digits]
18         if digits.startswith("978"):
19             variants.append("0" + digits[3:12])  # ISBN-13 → ISBN-10 → UPC-A
20         if digits.startswith("0"):
21             variants.append(digits[1:])      # 去掉首0
22
23     # 14位 → ITF-14，去前导0找更短格式
24     if len(digits) == 14:
25         stripped = digits.lstrip("0")
26         variants = [digits, stripped] if stripped else [digits]
27
28     return _dedupe(variants)    # 去重，保留顺序

接口 3 & 4 /eligibility
GET /eligibility/{asin}

接口意义

给定一个 ASIN，返回可转售合规性判断：是否满足 5 条业务规则，以及 ROI 预估。前端可渲染为逐条 CheckList。

5条合规规则（按优先级依次检查）

#字段条件含义
1referral_fee_pct> 0必须有有效佣金率（商品在 Amazon 有分类）
2rankrank ≤ 100,000 OR monthly_sold ≥ 100有真实需求（排名不太差 OR 月销够高则豁免排名检查）
3buybox≥ $10利润空间足够覆盖 FBA 费用
4amazon_pct≤ 80%非 Amazon 垄断，有第三方卖家竞争空间
5monthly_sold_minnull OR ≥ 100月销 1~99 之间视为需求不稳定

ROI 公式

payout  = buybox
         − referral_fee   (buybox × referral_fee_pct / 100)
         − fba_pick_pack  (fba_pick_pack_cents / 100)
         − storage        ($0.50 / 月)

roi     = 100 × (payout − supplier_cost) / supplier_cost

eligibility.py — 核心逻辑

 1 def compute_payout(buybox, referral_fee_pct, fba_pick_pack_cents) -> float:
 2     if buybox is None or buybox <= 0:
 3         return 0.0
 4     referral = buybox * (referral_fee_pct / 100)
 5     fba = (fba_pick_pack_cents or 0) / 100
 6     storage = 0.50
 7     return buybox - referral - fba - storage
 8
 9 def compute_roi(buybox, referral_pct, fba_cents, supplier_cost, n_items) -> Optional[float]:
10     if supplier_cost is None or supplier_cost <= 0:
11         return None
12     if buybox is None or buybox <= 0:      # 无价格 → 无法计算 ROI
13         return None
14     payout = compute_payout(buybox, referral_pct, fba_cents)
15     items = max(n_items or 1, 1)
16     total_cost = supplier_cost * items            # 多件装 × 件数
17     return 100 * (payout - total_cost) / total_cost
18
19 def check_eligibility(referral_fee_pct, sales_rank, monthly_sold, buybox, amazon_pct):
20     # 5条规则依次检查，返回第一个失败原因
21     if referral_fee_pct is None or referral_fee_pct <= 0:
22         return False, "referral_fee_pct"
23     rank_ok = sales_rank is not None and sales_rank <= 100_000
24     sold_ok = monthly_sold is not None and monthly_sold >= 100
25     if not (rank_ok or sold_ok):
26         return False, "rank"
27     if buybox is None or buybox < 10:
28         return False, "buybox"
29     if amazon_pct is None or amazon_pct > 80:
30         return False, "amazon_pct"
31     if monthly_sold is not None and monthly_sold < 100:
32         return False, "monthly_sold"
33     return True, None

main.py — /eligibility/batch

一次传入多个 ASIN（最多 ~100个），返回每个 ASIN 的 eligibility 结果。用于批量筛选、导出报告、AI 推荐后的二次确认。DB 中不存在的 ASIN 返回 filter_failed=not_found。

接口 5 /ask
POST /ask

接口意义

将自然语言问题转化为 SQL 查询，执行后用 LLM 生成人类可读回答。用户无需懂 SQL，直接问 "How many eligible ASINs?" 即可得到答案。

两阶段管道

"How many eligible?" → DeepSeek Step1
NL → SQL → SQL 校验
SELECT-only → Execute
SQLite → DeepSeek Step2
格式化回答 → answer + sql + rows

安全防护（三层）

层级机制拦截内容
1领域关键词过滤 _is_out_of_scope()weather, bitcoin, news 等域外问题直接拒绝，不调 LLM
2SQL 白名单 _validate_sql()仅允许 SELECT，拦截 DROP/DELETE/injection
3JSON 解析容错 _parse_sql_response()LLM 输出非 JSON 时不崩溃
